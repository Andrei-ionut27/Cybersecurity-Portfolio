# TCP SYN Flood Incident

## Overview

This project analyzes a TCP SYN Flood attack using a Wireshark network capture.

## Objective

- Analyze TCP traffic
- Identify a SYN Flood attack
- Understand the TCP Three-Way Handshake
- Document the incident

## Skills Demonstrated

- TCP
- TCP Three-Way Handshake
- SYN Flood Detection
- Wireshark
- Network Analysis
- Incident Reporting

## Files

- Incident-Report.pdf
- Wireshark-TCP_HTTP-Log.xlsx

## Outcome

The analysis identified a TCP SYN Flood attack. The attacker sent a large number of SYN packets without completing the TCP handshake, creating many half-open connections and preventing legitimate users from accessing the web server.
